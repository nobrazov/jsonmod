# JSONmod Language Reference

## 1. Базовый синтаксис

Файл `.jsonmod` — это **специальный JSON** с расширенными инструкциями. После обработки ядром преобразуется в строго валидный стандартный JSON.

### 1.1. Структура файла
```
[Объявления переменных]   # 0 или более строк вида: name = value
[JSON-структура]         # Валидный или полу-валидный JSON с директивами
```

### 1.2. Экранирование
| Последовательность | Результат |
|-------------------|-----------|
| `$$` | `$` |
| `\@` | `@` |
| `\"` | `"` (стандарт JSON) |
| `\\` | `\` (стандарт JSON) |

## 2. Директивы

Формат: `@ + тип + {селектор} + пробел + "путь_к_донору"`

> **Важно**: Директива — это **ключ без значения**. Двоеточие `:` после неё не ставится.

### 2.1. `@%` — Объект
| Синтаксис | Действие |
|-----------|----------|
| `@% "uri"` | Внедрить весь объект донора |
| `@%{"key"} "uri"` | Внедрить только ключ `key` из донора |
| `@%{"k1","k2"} "uri"` | Внедрить указанные ключи |

**Типы донора**:
- `{}` → ✅ Вставка
- `[{}]` → ✅ Unwrap (извлечение объекта)
- `[{},{}]` или иное → ❌ `ERR_TYPE_MISMATCH`

### 2.2. `@@` — Массив
| Синтаксис | Действие |
|-----------|----------|
| `@@ "uri"` | Слить весь массив донора |
| `@@{2} "uri"` | Взять элемент с индексом 2 |
| `@@{1..4} "uri"` | Диапазон элементов (включительно) |

**Типы донора**:
- `[...]` → ✅ Слияние
- `{}` или примитив → ❌ `ERR_TYPE_MISMATCH`

### 2.3. `@$` — Плоский список
| Синтаксис | Действие |
|-----------|----------|
| `@$ "uri"` | Инъекция пар из массива `["k1","v1","k2","v2",...]` |

**Требования к донору**:
- Строго одномерный массив чётной длины
- Элементы с чётными индексами — строки-ключи
- Элементы с нечётными индексами — значения любого типа

**Проверки**:
- Ключ уже есть в контейнере → ❌ `Duplicate key`
- Тип значения не совпадает с типом в контейнере → ❌ `ERR_TYPE_MISMATCH`

## 3. Переменные

### 3.1. Объявление
```
имя = значение
```
- Одна переменная на строку
- Объявляются **до** JSON-структуры
- Значения: строка `"..."`, число, `true`/`false`, `null`, массив `[...]`, объект `{...}`

### 3.2. Подстановка
- Синтаксис: `${имя}`
- Область: текущий контейнер `.jsonmod`
- Подставляется в:
  1. URI директив (до загрузки донора)
  2. Сырой контент донора (до парсинга)
- Рекурсивные ссылки запрещены

## 4. URI и протоколы

### 4.1. Формат
```
<scheme>://<path>
```

### 4.2. Поддерживаемые схемы
| Схема | Описание |
|-------|----------|
| `file://` | Локальная файловая система |

### 4.3. Разрешение пути (`file://`)
1. Отрезается префикс `file://`
2. `~` раскрывается в `$HOME` (`expandTilde`)
3. Относительный путь преобразуется в абсолютный (`absolutePath`)
4. Проверяется существование (`exists`)

### 4.4. Glob-маски (только для имени файла)
| Символ | Значение |
|--------|----------|
| `*` | Любая последовательность символов |
| `?` | Ровно один любой символ |
| `[abc]` | Один символ из набора |
| `[!abc]` | Один символ, не входящий в набор |

> **Примечание**: Порядок возврата файлов при `*` определяется файловой системой и не гарантируется детерминированным.

## 5. Ошибки

| Код | Описание |
|-----|----------|
| `ERR_INVALID_URI` | Отсутствует `://` в схеме |
| `ERR_UNSUPPORTED_PROTOCOL` | Неизвестная схема |
| `ERR_FILE_NOT_FOUND` | Файл не найден |
| `ERR_DIR_NOT_FOUND` | Директория не найдена (для glob) |
| `ERR_EMPTY_RESULT` | Глоб не нашёл файлов или пустой результат |
| `ERR_TYPE_MISMATCH` | Тип донора не соответствует ожидаемому |
| `Duplicate key` | Ключ из `@$` уже существует в контейнере |
| `ERR_KEY_NOT_FOUND` | Ключ из селектора `@%{"key"}` не найден в доноре |

### 5.1. Коды типов (для диагностики `ERR_TYPE_MISMATCH`)
| Код | Тип |
|-----|-----|
| 0 | `null` |
| 1 | `boolean` |
| 2 | `double` |
| 3 | `int` |
| 4 | `object` |
| 5 | `array` |
| 6 | `string` |
